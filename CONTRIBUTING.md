# Contributing to Recon Sentinel

Thank you for your interest in Recon Sentinel. This project is currently under proprietary license and is not accepting external contributions.

## Reporting Bugs

If you've found a bug, please open a GitHub issue with:

1. Steps to reproduce
2. Expected behavior
3. Actual behavior
4. Environment details (OS, Docker version, browser)

## Security Vulnerabilities

**Do not open a public issue for security vulnerabilities.** See [SECURITY.md](SECURITY.md) for our responsible disclosure policy.

## Feature Requests

Open a GitHub issue with the `feature-request` label. Include:

1. The problem you're trying to solve
2. Your proposed solution
3. Any alternatives you've considered

## Development Setup

See `docs/DEVELOPMENT.md` for local development instructions.

## Code Style

- **Python:** Follow PEP 8. Use `ruff` for linting.
- **TypeScript:** Follow the existing codebase patterns. No `any` types.
- **Commits:** Use conventional commit messages (`feat:`, `fix:`, `docs:`, `refactor:`, `test:`)
- **Tests:** All new features must include tests. Maintain >80% coverage.
